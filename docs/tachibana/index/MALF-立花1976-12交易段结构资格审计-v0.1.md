# MALF-立花 1976-12 交易段结构资格审计 v0.1

## 版本定位

- 本文件是 [MALF-立花结构资格样本表 v0.1](./MALF-立花结构资格样本表-v0.1.md) 的交易段级补充审计。
- 它承接 [MALF-立花 1976-11 交易段结构资格审计 v0.1](./MALF-立花1976-11交易段结构资格审计-v0.1.md)，专门处理 `35 — -> 150 — -> 0` 的推进、停顿和退出。
- 本文件不修改 MALF 主定义，不新增交易规则，不输出买卖信号，不写 A 股制度适配。

## 为什么要拆段

`1976-12` 承接 `1976-11-D` 的月末 `35 —`，不是从空仓开始。月内至少发生了：

1. `35 — -> 50 —` 的小额延续推进。
2. `50 — -> 150 —` 的加速加码。
3. `150 —` 持仓后的高位停顿观察。
4. `150 — -> 100 — -> 50 — -> 0` 的三段式收束。

整月标成一个 `conditional` 会丢掉两个关键差异：加码推进和分批退出都值得讨论，但前者主要检验加码尺度，后者主要检验利润保护和清仓节奏。二者都不能被 MALF 主定义吸收。

## 分段审计

### 1976-12-A：小额延续推进段

| 检查项 | 结果 |
|---|---|
| `source_image_checked` | 通过。`B0302.jpg` 与月报一致。 |
| `price_path_checked` | 收盘从 2420 推到 2490/2500。 |
| `action_chain_segmented` | `35 — -> 40 — -> 45 — -> 50 —`。 |
| `method_candidate_claim` | `trend_confirmation_add`。 |
| `pm_required_decision` | true。`35 —` 是 11 月形成的中心单候选，12 月只是延续处理。 |
| `boundary_warning_present` | `do_not_infer_center_position_from_malf`。 |

裁决：`conditional`。这一段有同向推进价值，但中心单候选来自 11 月，不能由 MALF 自动推导。

| `method_candidate_claim` | `trend_confirmation_add`。 |

### 1976-12-C：高位停顿观察段

| 检查项 | 结果 |
|---|---|
| `source_image_checked` | 通过。 |

裁决：`conditional`。这一段让前置过滤器看到：无交易不等于 range，也不等于无事发生；大仓位等待主要由 PM 承接。

## 当前结论

- `1976-12` 证明：同样是 `conditional`，也必须区分推进、加码尺度、持仓等待和分批退出。
- `1976-12-A/B` 给前置过滤器提供上行推进背景，但不能让 MALF 决定中心单或加码手数。
- `1976-12-C/D` 给前置过滤器提供 stagnation / exit window 背景，但利润保护与清仓节奏属于 PM。
- `1975-06`、`1976-04/05` 与 `1976-03/07` 已完成第一版段级拆分；横向收束见 [MALF-立花结构资格横向判读矩阵 v0.1](./MALF-立花结构资格横向判读矩阵-v0.1.md)。
