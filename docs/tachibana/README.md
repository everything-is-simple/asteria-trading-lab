## 来源纪律
- 事实：按月 JSON、交易谱截图、月度 PDF。
- 书中自述：OCR 总稿、操作法 PDF。
- 规则细节：用户挑出的 `Z:\market-life-cycle\立花义正交易法-part1-3` 页组优先；part4 交易谱说明用于校勘实战段的母单、锁单目的、加码和 1976-09-21 交易单位变化。
- 我们的抽象解释：为理解与回测服务，但不能冒充原文。
- MALF 映射判断：只做映射，不反向篡改交易事实。
- A 股适配判断：只用于中国 A 股改造版，不得反向混入原始立花回测。

## 当前攻坚顺序

第一优先级是把 MALF 作为立花法的前置认知过滤器做实，而不是先改 A 股交易规则。当前文档与代码共同固定以下顺序：

1. A 股最小接入包只能把真实候选样本送到 `structure_candidate`。
2. MALF snapshot 必须先通过质量与结构状态检查。
3. 前置过滤器输出 `rhythm_meaning / tachibana_applicability / qualification_rule_id`。
4. `rhythm_sample_row_gate` 与 `candidate_table_gate` 判断样本是否有资格进入结构资格表。
5. `method_pm_bridge_gate` 才允许 Method / PM 层解释动作与仓位节奏。
6. `backtest_input_gate` 才允许形成回测输入。
7. `cognitive_pipeline_gate` 全链路通过后，只允许启动 A 股制度约束审计；不直接生成 T+1、涨跌停或停牌规则。

这条链路的目的，是先回答“哪类股票、哪种结构状态下，立花法的仓位节奏才有意义”，再谈制度适配。
