# 2026-07-02 Formal Institution Rule Definition Write-Audit Design

**Status:** design-only
**Current baseline:** `formal_institution_rule_definition_persistence_package_prepared`
**Target entrypoint:** `audit_formal_institution_rule_definition_write_when_explicitly_requested`
**Scope:** after P8 persistence package preparation, before any explicit formal rule definition write confirmation gate

## 0. DecisionTrace

| Field | Content |
|---|---|
| `current_state` | P8 has prepared `formal_institution_rule_definition_persistence_package`. Current docs state no formal rule file has been generated, no real persistence has been performed, and trading / signal / backtest gates remain closed. |
| `candidate_routes` | `route_a`: enter P9 signal/backtest; `route_b`: add formal rule definition write-audit before any write; `route_c`: add broad writer, read gate, and P9 contract in one step. |
| `pruned_routes` | `route_a` is pruned because `trading_layer_read_allowed`, `signal_generation_allowed`, and `backtest_execution_allowed` are still false. `route_c` is pruned because it merges multiple gates and makes blocked states harder to audit. |
| `selected_route` | Add the smallest write-audit gate: `audit_formal_institution_rule_definition_write_when_explicitly_requested`. It audits whether the P8 package is eligible to proceed to a later explicit write confirmation gate. It does not write files. |
| `hard_boundaries` | `institution_rule_definition_allowed=True` remains rule-definition-only. `formal_institution_rule_definition_write_allowed=False`, `trading_layer_read_allowed=False`, `signal_generation_allowed=False`, `backtest_execution_allowed=False`, and real data-root writes remain closed. |
| `verification` | Add pass, blocked, forbidden-field, package identity, and hard-gate tests before implementation. Run focused tests, full unittest discovery, source-file-size contract, and `git diff --check`. |

## 1. Purpose

This gate verifies that a prepared formal institution rule definition persistence package is eligible to proceed to a later explicit write confirmation gate.

The pass result means only:

`ready_for_formal_institution_rule_definition_explicit_write_confirmation_gate`

It does not mean:

- a formal rule file exists;
- real persistence has happened;
- `formal_institution_rule_definition_write_allowed` becomes true;
- trading layer read is allowed;
- signal generation is allowed;
- backtest execution is allowed.

## 2. Proposed Entrypoint

`audit_formal_institution_rule_definition_write_when_explicitly_requested(...)`

Recommended inputs:

- `formal_rule_definition_persistence_package_report`
- `formal_rule_definition_write_audit_request`

The function returns an audit report only. It must not write to `Z:\asteria-trading-labs-data` or any other real data root.

## 3. Required Input Semantics

`formal_rule_definition_persistence_package_report` must prove:

- `audit_id == "formal_institution_rule_definition_persistence_package_v0.1"`
- `report_id` is present and was generated by `prepare_formal_institution_rule_definition_persistence_package_when_explicitly_requested`;
- `package_id` is present;
- `package_version` is present;
- `formal_institution_rule_definition_persistence_package_result == "pass"`;
- `formal_institution_rule_definition_persistence_package_status == "formal_institution_rule_definition_persistence_package_prepared"`;
- `formal_institution_rule_definition_persistence_package_prepared == True`;
- `formal_institution_rule_definition_persistence_performed == False`;
- `formal_institution_rule_definition_write_allowed` is absent or false;
- `institution_rule_definition_allowed == True`;
- `trading_layer_read_allowed == False`;
- `signal_generation_allowed == False`;
- `backtest_execution_allowed == False`;
- package payload exists;
- field contract is complete;
- boundary status is clean;
- evidence status is ready.

`formal_rule_definition_write_audit_request` must prove:

- `write_audit_requested == True`;
- `write_audit_scope == "formal_institution_rule_definition"`;
- `target_kind == "formal_rule_definition_file"`;
- `target_package_id` matches the P8 report `package_id`;
- `target_package_version` matches the P8 report `package_version`;
- `real_data_root_write_confirmed == False`;
- `manual_confirmation_required == True`;
- `no_trading_no_signal_no_backtest_acknowledged == True`.

## 4. Pass Output

A pass report should include:

- `audit_id = "formal_institution_rule_definition_write_audit_v0.1"`
- `formal_institution_rule_definition_write_audit_result = "pass"`
- `formal_institution_rule_definition_write_audit_status = "ready_for_formal_institution_rule_definition_explicit_write_confirmation_gate"`
- `formal_institution_rule_definition_write_allowed = False`
- `formal_institution_rule_definition_persistence_performed = False`
- `source_package_id = <package_id>`
- `source_package_version = <package_version>`
- `institution_rule_definition_allowed = True`
- `institution_rule_definition_scope = "rule-definition-only"`
- `trading_layer_read_allowed = False`
- `signal_generation_allowed = False`
- `backtest_execution_allowed = False`
- `next_gate = "gate:formal_institution_rule_definition_explicit_write_confirmation"`

`formal_institution_rule_definition_write_allowed` is intentionally false in the pass output. Pass means eligible to proceed to the explicit write confirmation gate, not that write is authorized.

`next_gate` is only a named future gate. It must not be executed by this function.

## 5. Blocked Scenarios

The function must return blocked when:

- P8 report is missing;
- P8 report `audit_id` is wrong;
- P8 report `report_id` is missing;
- P8 report `package_id` is missing;
- P8 report `package_version` is missing;
- P8 result is not pass;
- P8 status is not `formal_institution_rule_definition_persistence_package_prepared`;
- persistence package is not prepared;
- persistence has already been performed;
- input report contains `formal_institution_rule_definition_write_allowed == True`;
- package payload is missing;
- field contract is not complete;
- boundary status is not clean;
- evidence status is not ready;
- `institution_rule_definition_allowed` is not true;
- `institution_rule_definition_scope` is present and is not `rule-definition-only`;
- `write_audit_requested` is missing or false;
- `write_audit_scope` is not formal institution rule definition;
- target kind is wrong or missing;
- request `target_package_id` does not match P8 report `package_id`;
- request `target_package_version` does not match P8 report `package_version`;
- request claims real data-root write is already confirmed;
- request does not acknowledge manual confirmation requirement;
- request does not acknowledge no trading, no signal, no backtest;
- any hard gate is already open;
- any forbidden trading, position, signal, execution, or backtest field appears anywhere in inputs.

## 6. Staleness and Idempotency

First implementation must record the staleness decision explicitly.

Recommended first-version rule:

- If `package_created_at` is present, preserve it in the report but do not enforce a max age yet.
- If `package_created_at` is missing, do not block solely for missing staleness metadata in v0.1.
- Add `package_staleness_policy = "not_enforced_v0.1"` to the pass report.

First implementation must also record the idempotency decision explicitly.

Recommended first-version rule:

- Re-running the write audit for the same `package_id` and `package_version` is idempotent when inputs match.
- If an input request claims a different package identity or claims previous write completion, block.
- Add `write_audit_idempotency_policy = "same_package_identity_is_idempotent_v0.1"` to the pass report.

## 7. Forbidden Fields

Inputs and output payloads must not contain:

- `buy_signal`
- `sell_signal`
- `trade_accept`
- `trade_reject`
- `trade_defer`
- `signal_decision`
- `target_position`
- `position_size`
- `order_instruction`
- `pnl`
- `backtest_result`
- `ashare_t1_action`
- `limit_up_strategy`
- `limit_down_strategy`
- `rhythm_meaning_override`
- `tachibana_applicability_override`

If a forbidden field is found, return a sanitized issue code:

`formal_institution_rule_definition_write_audit_forbidden_output_field_present`

Do not echo the forbidden field name into report details.

## 8. Non-Goals

This spec does not implement:

- formal rule file writing;
- real data-root persistence;
- trading layer read;
- signal generation;
- position sizing;
- order generation;
- backtest execution;
- P9 implementation.

## 9. Test Requirements

Add tests covering:

- pass: valid P8 package plus explicit write-audit request;
- blocked: missing P8 report;
- blocked: failed P8 report;
- blocked: package not prepared;
- blocked: persistence already performed;
- blocked: missing P8 `report_id`;
- blocked: missing P8 `package_id` or `package_version`;
- blocked: request `target_package_id` mismatch;
- blocked: request `target_package_version` mismatch;
- blocked: input report contains `formal_institution_rule_definition_write_allowed == True`;
- blocked: input report contains `formal_institution_rule_definition_write_allowed == "true"`;
- blocked: missing write-audit request;
- blocked: request tries to confirm real data-root write early;
- blocked: forbidden field without echo;
- hard gates: pass and blocked outputs keep trading / signal / backtest false;
- idempotency: same package identity returns a stable pass report.
