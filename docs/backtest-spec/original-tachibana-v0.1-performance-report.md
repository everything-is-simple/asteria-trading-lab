# 原始立花法 v0.1 正规绩效回测报告

## 结论摘要

在 v0.1 绩效口径下，1975-1976 两年样本整体盈利。主要盈利来自 1976-10 至 1976-11 的大规模多头买入并在高位卖出清仓段。

本报告已按立花交易谱记号修正方向：`—5` 表示买入 5 张，`5—` 表示卖出 5 张；未平仓栏中横杠左侧为空头库存，横杠右侧为多头库存。

## 回测假设

| 项 | 口径 |
|---|---|
| 标的 | Pioneer 1975-1976 月度交易谱 |
| 交易周期 | 日线 |
| 样本区间 | 1975-01-01 至 1976-12-31 |
| 交易单位 | v0.1 算术口径为“价格点 × 记录手数/单位”；part4 记录 PIONEER 自 1976-09-21 起交易单位改为 100 股，本报告暂不换算真实股数金额 |
| 判断依据 | 上一有效交易日收盘价，即早晨从报纸取得的已知信息 |
| 下单时点 | 下一交易日开盘前电话下市价单 |
| 成交价格 | `trade_price`，解释为成交日开盘价 |
| 盯市价格 | 成交后的 `close_price`；若当日无收盘价，则使用 `trade_price` 或上一有效价格 |
| 成本 | 手续费 0，滑点 0 |
| 资金基准 | 样本内最大总名义暴露：472000 点手 |
| 无风险利率 | 0 |
| 交易样本 | 已清仓的 `segment_id` |
| 盈亏算法 | 按每笔成交价、成交手数和方向，使用多头/空头均价成本逐笔累加，按交易段汇总已实现盈亏 |

时点修正：交易行的同日 `close_price` 不能作为该行下单依据，只能作为成交后的盘后记录和盯市价格。本报告的 PnL 使用 `trade_price` 成交，因此该修正不改变当前绩效数值，但会约束后续信号生成和 MALF 接入，避免未来函数。

单位修正：part1-3 支持早期 1000 股口径；part4 记录 PIONEER 自 1976-09-21 起交易单位改为 100 股。当前 v0.1 仍以“点手”为展示单位，若换算到真实股数口径，成交额和 PnL 需要按日期/交易单位表处理，不能全样本统一乘 1000。

## 风险观察

- S013 是全样本主要利润来源，`—2 -> —2 -> —20 -> —50 -> —102` 累积买入至多头 200 张，随后 `200 —` 在 2340 附近卖出清仓，单段盈利约 41140。
- 胜率 66.67%，且平均盈利约为平均亏损的 7.72 倍，导致期望值显著为正。
- Profit Factor 为 15.45，说明 v0.1 当前方向解释与交易段切分下，盈利段显著覆盖亏损段。
- 加码尺度警戒集中在 1976-11 与 1976-12，符合 PM 规格中对极端加码的风险标注。

## 当前限制

- 本报告不含手续费、滑点、融资成本、保证金约束。
- 本报告未接入 MALF 结构背景，也未使用人工确认的中心单/锁单标注。
- 交易信号必须使用 `decision_basis_date` 及其以前的数据，不得使用执行日同日收盘价。
- `center_side` 和多空方向已按当前校正后的交易谱记号解释；若未来有新的原文证据修正记号语义，绩效指标必须重算。
- 本报告不代表 A 股适配版表现。
