## 本轮修正结论

昨天的回测工作可以保留为“part4 交易谱回放原型”，但不能称为完整定版。part1-3 说明的是更底层的操作系统：记录法、工具法、训练法、下单纪律、分批基准、锁单目的、中心单/母单处理。这些内容必须成为 v0.1 规格的上位约束。

## 记录方式

立花的交易记录横杠不是数学负号，而是方向标记。横写记录表的规则如下：

| 写法 | 书中含义 | 回测字段 |
|---|---|---|
| `—1` / `—1000` | 买 1 记录单位；书中早期也以 1000 股口径说明。 | `buy` 或 `gross_long = 1` |
| `1—` / `1000—` | 卖 1 记录单位；书中早期也以 1000 股口径说明。 | `sell` 或 `gross_short = 1` |
| `[2—5]` | 同时持有卖单 2 记录单位与买单 5 记录单位；真实股数需按当日交易单位换算。 | `gross_short = 2`, `gross_long = 5` |
| `—2 (600日圆)` | 以 600 日圆买 2 记录单位；真实股数需按当日交易单位换算。 | `side=buy`, `qty=2`, `price=600` |

因此：

- 横杠右边是买单/多头库存。
- 横杠左边是卖单/空头库存。
- `trade_raw` 与 `position_raw` 必须分开解释；同一符号可表示当日买卖，也可表示未平仓。
- 不能把 `10—2` 压成净空 8 记录单位；它是空 10 记录单位、多 2 记录单位的双侧库存事实。

## 交易单位

part1-3 明确说早期股票交易单位进入 `1,000股` 口径；part4 交易谱说明 PIONEER 自 `1976-09-21` 起交易单位改为 `100股`。

当前回测保留两个层次：

| 层次 | 说明 |
|---|---|
| `point_unit` | 当前 v0.1 使用的计算层：价格点 × 记录单位。 |
| `share_pnl` | 资金层：按日期/交易单位表将 `point_unit` 换算为真实股数金额，后续资金版再展开。 |

胜率、Profit Factor、盈亏比、夏普率等比率指标不会因为股数乘数改变；金额类指标必须标明是否已按日期换算真实股数。

## 下单与价格纪律

立花不是盘中看盘下单，也不是看当天收盘后当天成交。他的基本流程是：

| 步骤 | 规则 |
|---|---|
| 盘后/早晨记录 | 从报纸抄前一交易日收盘价。 |
| 下单判断 | 根据已看到的收盘价、价位表、部位卡和图形作判断。 |
| 下单时间 | 开盘前，通常 9 点前。 |
| 下单方式 | 市价单为基本纪律；限价下单被视为容易带来贪婪和执行失败。 |
| 成交价记录 | 第二天从报纸补抄成交价；因使用市价单，成交价按开盘价理解。 |
| 交易日表格 | 下单当天，成交价和收盘价栏可能为空，第二天再补。 |

因此回测不得使用交易行同日收盘价解释当日下单原因。交易行的 `close_price` 只能用于成交后的记录和盯市。

## 工具系统

part1-3 中的“方法”不是单纯买卖信号，而是一套工具系统：

| 工具 | 作用 |
|---|---|
| 价位表 | 排除视觉模糊，培养波动感；股票后期重视收盘价表。 |
| 部位管理卡 | 管理留仓单、平均价和下一步下单计划；空单左边、多单右边。 |
| 买卖管理表 | 记录买卖、成交价、数量、盈亏和交易经过。 |
| 资金管理表 | 区分手头资金、存款、利润和损失。 |
| 图形 | 用于观察走势，不是为了堆叠多周期指标。 |
| 自我约束清单 | 把具体禁令写在桌前，防止重复重大错误。 |

对回测系统而言，价位表/部位卡/买卖管理表分别对应：

- `price_table_state`
- `inventory_state`
- `trade_ledger`
- `manual_rule_notes`

## 分批与训练基准

立花的基础训练不是预测，而是固定模式反复执行。

| 规则 | 说明 |
|---|---|
| 三分之一攤平 | 早期基础训练，以分三批进场为核心。 |
| 不平均分批 | 从 `1-2-3` 到 `2-3-5`，重视让平均价对自己有利。 |
| 分批出场 | 基本是分三批进场，分三批或一批平仓。 |
| 再次进场 | 再次买入价应低于前一轮分批出场中的高价；重在节奏感。 |
| 不追行情 | 大涨第二天追买、崩盘追空都属于应克服的冲动。 |
| 顺向只限测试 | 错失进场时可用顺向测试单，但不能作为大仓加码。 |

这意味着自动回测只能识别“分批事实”，不能仅凭价格自动判定“正确分批”。分批的训练目的、节奏感和执行质量需要人工标注或结构背景辅助。

## 加码与中心单

part1-3 对加码的约束比一般“顺势加仓”更严：

| 概念 | 校勘后定义 |
|---|---|
| 中心单/母单 | 交易者愿意维持、保护、围绕其调整的核心库存。 |
| 加码单 | 围绕中心单/母单追加的攻击性库存，不应让加码陷入更不利位置。 |
| 加码纪律 | 母单进场后的加码方法不必改成追涨杀跌；仍以逆向/回档/反弹方式更稳。 |
| 尺度 | 测试单和加码单的数量必须受总仓、心理压力和平均价控制。 |

当前 PM 中的 `center_position` 与 `add_on` 只能作为候选。最终哪部分是母单、中心单、加码单，必须结合书页解说和人工标注。

## 锁单的重新定义

昨天的文档容易把锁单理解为“利润保护”。part1-3 说明，锁单至少有多种用途：

| 类型 | 说明 |
|---|---|
| 测试单性质的反向单 | 用小反向单测试行情和自己的感觉。 |
| 维持母单的反向锁单 | 为了保留母单，同时获得处理走弱行情的线索。 |
| 利润确保型锁单 | 用反向单替代直接平掉一部分母单，保住继续参与行情的能力。 |
| 反向操作的过渡 | 若原母单平掉，反向锁单可转化为获利为目的的新投机单。 |
| 失败补救型锁单 | 书中承认有时失败时也用锁单，但这不是理想习惯。 |

因此，`gross_long > 0 && gross_short > 0` 只能生成 `lock_candidate`。它不能自动等于：

- 正式锁单
- 利润保护
- 价差交易
- 反手
- 错误补救

必须新增或保留 `lock_purpose` 人工标注字段，候选值包括：

`reverse_probe / mother_position_protection / profit_lock / reversal_bridge / failure_repair / spread_like / unknown`

## 单一标的与比较标的

立花强调长期专心操作熟悉标的。不同股票的锁单/价差会分散精力；SONY 与日本电气案例显示，即使总体赚钱，也可能伤害主标的操作。

对 A 股适配的含义是：第一版不应做全市场信号扫描，应先选择少数熟悉、流动性好、交易连续性强的标的，训练固定记录和分批执行。
